Maximum input size

The maximum width and height to which all input images will be resized before feeding to the neural network, while maintaining their original aspect ratio. Generally, a smaller input size is acceptable for tracking a single keypoint or when the animal is large relative to the arena. A larger input size is preferable when tracking multiple keypoints or when the animal is small relative to the arena, but will be slower.

Number of channels

The channel capacity of the neural network. A smaller capacity will result in a very fast network, but may not provide sufficient accuracy.

Number of blocks

The number of blocks through which data will flow within the neural network. Fewer blocks provides a shorter path through the network and is thus faster. More blocks gives the network more freedom to learn complex transformations of the data from input to output, but may make learning less efficient. The default setting is generally a safe choice.

Batch size

The number of images that will be fed to the neural network before each learning consolidation step. A smaller batch size leads to more rapid learning, but may be less stable.

Learning rate

The magnitude of the learning updates. Small learning rates may take too long to train but are stable, whereas high learning rates train quickly but are unstable. The ideal learning rate is somewhere between those two extremes, but its exact value depends on the other training settings and on the dataset.
